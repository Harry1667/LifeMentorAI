claude# 產品需求文檔 (PRD)：人生導師 AI 系統 (Life Mentor AI)

## 1. 產品概述
**產品名稱**：人生導師 AI 系統 (暫定)
**產品願景**：透過擷取歷史與現代偉人、專家的真實文獻與理論，構建具有「靈魂（人格）」與「思想（理論）」的 AI Agent 群組。系統能透過深度對話理解使用者的真實困境，並藉由多個導師與理論 Agent 的內部探討，為使用者提供高度客製化、可追蹤進度的人生解決方案，陪伴使用者成長。
**目標受眾**：希望改善生活節奏、提升學習效率、尋求人生或財務建議的年輕人（如：居住在台灣的大學生）。

## 2. 核心功能模組

### 2.1 導師人格構建模組 (Persona Engine - 靈魂)
* **功能描述**：賦予 AI 導師特定人物的人格特質與背景知識。
* **運作機制**：
    * 支援輸入特定人物名稱（在世或已故皆可）。
    * 系統自動觸發網路爬蟲與檢索，抓取該人物的傳記、文獻、公開談話、著作、訪談紀錄等資料。
    * 將資料向量化並儲存，形成該導師的專屬知識庫 (RAG)，使其發言風格與價值觀高度貼合歷史真實人物。

### 2.2 理論載入模組 (Theory Engine - 思想)
* **功能描述**：將抽象理論具象化為獨立的思考邏輯，作為討論時的輔助或主導思想。
* **運作機制**：
    * 使用者可手動輸入特定「理論名稱」（例如：原子習慣、第一性原理、斯多葛學派）。
    * 系統自動聯網檢索該理論的精確定義、應用場景、優缺點與詳細說明。
    * 該理論轉化為一個「思想 Agent」，在討論過程中負責檢視解決方案是否符合該理論的框架。

### 2.3 深度釐清與多 Agent 協同討論機制 (Multi-Agent Collaboration)
* **功能描述**：不直接給出罐頭建議，而是透過「釐清」與「腦力激盪」來產出最佳解。
* **運作機制**：
    * **探底詢問**：當使用者提出困難時，系統（或主導師）會先向使用者提出 1-3 個反問，充分理解問題的背景與細節。
    * **圓桌會議**：確認問題全貌後，多位「導師 Agent」與「理論 Agent」會在後台（或前台展示對話過程）進行交叉討論。例如：富蘭克林（導師）與《原子習慣》（理論）共同針對「如何建立早起規律」進行辯論與融合，最後產出綜合建議。

### 2.4 行動反饋與追蹤系統 (Action & Feedback Tracker)
* **功能描述**：將建議轉化為可執行的任務，並透過使用者的真實反饋來優化未來的建議。
* **運作機制**：
    * 針對討論產出的具體解決方案，使用者可以標記三種狀態：
        1.  **接受 (Accepted)**：認同並準備開始執行。
        2.  **執行百分比 (Progress %)**：例如標註「已完成 50%」，系統可針對未完成的部分提供額外鼓勵或微調方案。
        3.  **拒絕 (Rejected)**：使用者覺得不可行。需附帶或透過對話說明拒絕原因（例如：太花時間、不符合目前作息）。
    * 這些反饋會成為新一輪對話的輸入，系統會即時修正溝通策略與方案。

### 2.5 伴隨式長期記憶與人格重塑 (Long-term Memory & Alignment)
* **功能描述**：系統隨時間推移越來越了解使用者。
* **運作機制**：
    * 系統建立使用者的專屬檔案（User Profile），透過每次的提問、反饋與對話，自動更新使用者的價值觀、痛點、優勢與作息習慣。
    * **契合度分析**：導師會主動評估使用者目前的狀態是否符合某個已載入的「理論概念」（例如：「你最近的專注力下降，似乎違背了我們之前討論的心流理論，需要調整嗎？」）。
    * 讓 AI 導師從「單向建議者」變成「深知你為人的老朋友」。

## 3. 使用者流程 (User Flow)

1.  **初始化設定**：使用者輸入基本資訊（大學生、居住地等）。
2.  **創建智囊團**：輸入想要召喚的導師名單（如：查理·蒙格、蘇格拉底）與想應用的理論（如：番茄鐘工作法）。
3.  **提出困境**：使用者輸入近期遇到的生活問題。
4.  **AI 釐清問題**：系統反問細節，確認問題的邊界。
5.  **協同討論**：導師與理論 Agent 針對問題進行推演，產出具體解決方案。
6.  **執行與反饋**：使用者給予方案「接受 / 執行進度 / 拒絕」的反饋。系統記錄並更新使用者的人格記憶庫。

## 4. 典型使用情境 (User Scenarios)

* **情境：重建生活秩序與專注力**
    * **使用者輸入**：「我最近作息很亂，生物鐘日夜顛倒。我想要讓生活更有序，健康睡眠，並且在學業上能更專心、學習更快。」
    * **系統釐清**：「了解。請問你目前大概幾點入睡？是因為學業壓力還是滑手機導致晚睡？你目前最想優先改善的是『睡眠規律』還是『學習專注度』？」
    * **討論過程**：
        * *導師 A (例如：富蘭克林)* 強調紀律與作息表的建立。
        * *理論 B (例如：微習慣理論)* 建議不要一次大幅調整，每天提早 15 分鐘入睡即可。
        * *導師 C (例如：費曼)* 針對學習給予建議，提出透過教學相長來提升專注與學習速度。
    * **方案產出**：結合三人意見，給出一套無痛調整生物鐘與費曼學習法的執行計畫。
    * **使用者反饋**：使用者將「早起計畫」標示為「執行 30%（昨天還是稍微晚睡了）」，將「費曼學習法」標示為「拒絕（目前沒有可以討論的對象）」。
    * **系統應對**：系統記憶使用者目前缺乏共學對象，未來不再推薦需要多人協作的學習法；並針對早起計畫給予微調方案。

## 5. 系統架構建議 (技術視角)

* **前端介面**：支援對話框 (Chat UI) 與任務看板 (Task Board，用於管理進度與狀態)。
* **Agent 框架**：可考慮使用 LangChain 或 AutoGen 來實作多智能體對話與協作邏輯。
* **知識庫與記憶**：
    * **外部資料 (導師/理論)**：網路爬蟲 + 向量資料庫 (Vector Database, 如 Pinecone, Chroma) 實現 RAG (檢索增強生成)。
    * **長期記憶 (使用者)**：Mem0 或自行架設基於圖資料庫 (Graph Database) 或關聯式資料庫的 User Profile 狀態機，記錄拒絕/接受的歷史與性格標籤。
